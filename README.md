# Инструкция по ненапряжной настройке telemt

## Быстрый старт

1. Обновить пакеты:
```
sudo apt-get update
```

2. Установить `make`:
```
sudo apt-get install -y make
```

3. Установить `docker`:
```
make install
```

> После этого шага нужно переподключиться к SSH-сессии, чтобы группа `docker` применилась.

4. Создать конфиг (автоматически подставит IP и сгенерирует секрет):
```
make template
```

5. Запустить контейнер:
```
make up
```

6. Подождать, пока всё инициализируется, затем получить ссылку на прокси:
```
make grep
```

---

## Все команды

### Управление контейнером

| Команда | Описание |
|---|---|
| `make up` | Запустить контейнер |
| `make down` | Остановить контейнер |
| `make logs` | Следить за логами |
| `make grep` | Вывести строки с прокси-ссылками (EE-TLS) |
| `make amount` | Показать количество активных TCP-соединений |

### Лимит трафика (через `tc`)

По умолчанию используется интерфейс `eth0`, лимит `2mbps`. Можно переопределить:
```
make limit DEV=ens3 RATE=5mbps
```

| Команда | Описание |
|---|---|
| `make limit` | Применить ограничение трафика (SSH без лимита, остальное — 2mbps) |
| `make unlimit` | Снять ограничение |
| `make relimit` | Переприменить (если правила уже есть) |
| `make status` | Показать текущие правила `tc` |

## Обновление прокси

```
make down
docker compose pull
make up
```

## Изменить секрет

```
make template   # сгенерирует новый секрет и перезапишет telemt.toml
make down
make up
```
